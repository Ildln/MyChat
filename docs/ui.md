# UI

## Что уже умеет текущий UI

Текущий UI в `app/static/index.html` предназначен для ручного тестирования backend внутри проекта MyChat.

Сейчас интерфейс умеет:

- регистрировать пользователя через `POST /auth/register`
- выполнять логин через `POST /auth/login`
- хранить `access_token` в `localStorage`
- получать текущего пользователя через `GET /auth/me`
- отправлять заявку в друзья через `POST /friends/requests`
- загружать входящие и исходящие заявки
- принимать и отклонять входящие заявки
- загружать список друзей
- создавать `direct chat` с другом через `POST /chats/direct`
- загружать список чатов через `GET /chats`

## Ограничения текущего этапа

Этот UI пока намеренно простой и нужен в первую очередь для ручной проверки backend.

Ограничения:

- нет окна переписки и отправки сообщений в рамках новой chat-модели
- нет групповых чатов
- нет сложной навигации и маршрутизации
- нет автоматического обновления данных
- список друзей, заявок и чатов загружается по кнопкам
- интерфейс не претендует на финальный дизайн приложения

## Текущий сценарий использования

1. зарегистрировать или залогинить пользователя
2. сохранить токен в интерфейсе
3. проверить `GET /auth/me`
4. отправить заявку в друзья
5. под другим пользователем принять заявку
6. загрузить список друзей
7. создать `direct chat`
8. загрузить список чатов
