# UI

## Новая структура интерфейса

Текущий UI в `app/static/index.html` разделён на два основных сценария:

1. экран авторизации
2. основной экран приложения после входа

Интерфейс по-прежнему состоит из трёх колонок:

- слева чаты
- по центру выбранный диалог
- справа друзья, заявки и действия

## Экран авторизации

Если токена нет в `localStorage`, пользователь видит только auth screen.

На этом экране есть:

- отдельная вкладка входа
- отдельная вкладка регистрации
- сообщения об ошибках и успешных действиях
- автоматический переход в основной экран после успешного входа или регистрации

## Основной экран приложения

Если токен есть в `localStorage`, пользователь сразу попадает в основной экран приложения.

В шапке доступны:

- информация о текущем пользователе
- обновление профиля через `GET /auth/me`
- logout, который очищает токен и возвращает на auth screen

## Левая колонка

Левая колонка отвечает за direct chats.

Сейчас она умеет:

- создавать `direct chat` через `POST /chats/direct`
- загружать список чатов через `GET /chats`
- выбирать нужный чат для открытия в центральной колонке

## Центральная колонка

Центральная колонка теперь поддерживает переписку внутри выбранного чата с realtime-обновлением.

Что происходит при выборе чата:

- сохраняется `selectedChatId`
- загружается история через `GET /chats/{chat_id}/messages`
- после этого открывается WebSocket на `/ws/chats/{chat_id}?token=...`
- в центре отображаются сообщения выбранного чата

Что есть в центре сейчас:

- заголовок выбранного чата
- информация об участниках
- список сообщений
- пустое состояние, если чат не выбран
- пустое состояние, если в чате ещё нет сообщений
- поле ввода сообщения
- кнопка отправки

Отправка работает так:

- если WebSocket активен, сообщение отправляется через realtime-канал
- если WebSocket недоступен, UI использует `POST /chats/{chat_id}/messages` как fallback

После успешной отправки:

- поле ввода очищается
- при активном WebSocket новое сообщение появляется сразу
- при REST fallback история сообщений загружается заново

Дополнительно:

- при переключении между чатами старое socket-подключение закрывается
- для выбранного чата показывается состояние соединения
- UI не дублирует сообщения, если они уже были показаны в центре

## Правая колонка

Правая колонка отвечает за друзей и заявки.

Сейчас она умеет:

- отправлять заявку в друзья
- загружать входящие заявки
- загружать исходящие заявки
- принимать входящие заявки
- отклонять входящие заявки
- загружать список друзей
- показывать состояние токена, текущего пользователя и последнего запроса

## Ограничения текущего этапа

На этом этапе UI всё ещё остаётся простым и ориентированным на ручную проверку backend.

Ограничения:

- нет групповых чатов
- нет маршрутизации между страницами
- нет отдельного экрана настроек
- история сначала загружается через REST, а новые сообщения приходят через WebSocket
- при проблемах с socket UI автоматически остаётся в обычном REST-режиме
