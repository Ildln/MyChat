# Chats

## Текущий шаг chats flow

На этом шаге в MyChat реализовано только создание личного `direct chat` 1 на 1 между друзьями.

Пользователь:

1. проходит аутентификацию
2. должен иметь подтвержденную дружбу с другим пользователем
3. вызывает `POST /chats/direct`
4. получает существующий или новый личный чат с двумя участниками

Групповые чаты и обмен сообщениями через новую chat-модель будут добавлены на следующих шагах.

## POST /chats/direct

Создает личный `direct chat` между текущим пользователем и его другом.

Для запроса нужен Bearer token:

```http
Authorization: Bearer <jwt>
```

Тело запроса:

```json
{
  "user_id": 2
}
```

Пример успешного ответа:

```json
{
  "id": 1,
  "type": "direct",
  "created_at": "2026-03-09T12:00:00Z",
  "members": [
    {
      "id": 1,
      "username": "alice"
    },
    {
      "id": 2,
      "username": "bob"
    }
  ]
}
```

## Бизнес-правила текущего шага

- endpoint доступен только авторизованному пользователю
- direct chat можно создать только с пользователем из списка друзей
- нельзя создать direct chat с самим собой
- нельзя создать direct chat с несуществующим пользователем
- если direct chat между той же парой уже существует, новый чат не создается
- в ответе возвращается только безопасная информация о чате и участниках
- на этом шаге поддерживается только тип чата `direct`
